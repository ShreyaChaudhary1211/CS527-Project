59    /**
60    * Given a set of ontologies, this composite change will convert all property
61    * assertion axioms whose subject is a 'punned' individual (i.e. an individual
62    * that shares a name with a class), removes these axioms and replaces them with
63    * annotations on the class that shares the same name as the individual. For
64    * example for a class A and an individual A, the data property assertion
65    * hasX(A, "Val") would be converted to an entity annotation on the class A with
66    * an annotation URI of "hasX" and a value of "Val".<br>
67    * This composite change supports refactoring an ontology where punning was used
68    * to simulate annotations on a class rather than using actual annotations on a
69    * class.
70    *
71    * @author Matthew Horridge, The University Of Manchester, Bio-Health
